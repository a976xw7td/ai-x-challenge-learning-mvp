# P1a 小班试运行验收 Checklist

> 目标：让一个真实小班（≤30 人）走通 发布→提交→AI 初评→教师终评→通知回流 全链路。
> 每个步骤标注操作人、预期结果和通过条件。

## 前置条件

- [ ] 飞书 Students 表至少导入 1 名真实学生（含 `学生ID`、`姓名`、`feishu_open_id`）
- [ ] `TEACHER_IDS` 已设为正式教师 ID 列表（逗号分隔）
- [ ] `FEISHU_CLASS_CHAT_ID` 已配置（班级群公告用，可选）
- [ ] `npx next build` 编译零错误
- [ ] Dev server 可正常启动（`npx next dev -p 3333`）

## 验收流程

### 第 1 步：学生登录（T9）
| # | 操作 | 预期结果 | 通过？ |
|---|------|---------|--------|
| 1.1 | 浏览器打开 `/login` | 显示学生 ID + 姓名输入框 | ☐ |
| 1.2 | 输入不存在的学生 ID 提交 | 提示"学生 ID 不存在或未导入系统" | ☐ |
| 1.3 | 输入存在的学生 ID + 正确姓名 | 跳转到 `/dashboard`，右上角显示姓名 | ☐ |
| 1.4 | 输入存在的学生 ID + 错误姓名 | 提示"姓名不匹配" | ☐ |

### 第 2 步：查看 Challenge（T10/T14）
| # | 操作 | 预期结果 | 通过？ |
|---|------|---------|--------|
| 2.1 | 进入 `/dashboard` | 显示已发布的 Challenge 列表（来自飞书真实数据） | ☐ |
| 2.2 | 点击某个 Challenge | 跳转 `/challenges/[id]`，显示完整信息（标题/简介/目标/交付物/评分标准/截止时间） | ☐ |

### 第 3 步：提交项目（T7/T8/T10）
| # | 操作 | 预期结果 | 通过？ |
|---|------|---------|--------|
| 3.1 | 在 Challenge 详情页填写提交表单 | 所有必填项（项目名称/简介/GitHub 链接/AAR/自评） | ☐ |
| 3.2 | 填入不存在的 GitHub 仓库 | 提交成功但 GitHub 检查返回"仓库不存在" | ☐ |
| 3.3 | 提交成功 | 返回 AI 初评结果（5 维度打分 + 总评语），含完整 auditTrail | ☐ |
| 3.4 | 查看飞书 Submissions 表 | 新增 1 条记录，status 已更新 | ☐ |
| 3.5 | 查看飞书 AuditLogs 表 | 新增 10+ 条审计记录（含提交全流程每一步） | ☐ |
| 3.6 | 查看飞书 Evaluations 表 | 新增 1 条 AI 初评记录（evaluator_type=ai） | ☐ |
| 3.7 | 查看飞书 Portfolio 表 | 新增 1 条作品集记录 | ☐ |

### 第 4 步：教师评审（T11）
| # | 操作 | 预期结果 | 通过？ |
|---|------|---------|--------|
| 4.1 | 教师登录（`TEACHER_IDS` 中的 ID） | role=teacher | ☐ |
| 4.2 | 进入 `/teacher` 页面 | 显示所有学生的提交列表 | ☐ |
| 4.3 | 在某条提交上点击"通过" | 弹出评分/评语对话框 | ☐ |
| 4.4 | 填入分数+评语，确认 | 提交记录状态更新为 `accepted`，task_state=`COMPLETED` | ☐ |
| 4.5 | 查看飞书 Evaluations 表 | 新增 1 条教师评审记录 | ☐ |
| 4.6 | 查看飞书 AuditLogs 表 | 新增评审流程审计记录 | ☐ |

### 第 5 步：通知验证（T8/T16）
| # | 操作 | 预期结果 | 通过？ |
|---|------|---------|--------|
| 5.1 | 学生提交成功 | 学生飞书收到 Bot 私聊：提交成功 + AI 初评分数 | ☐ |
| 5.2 | 教师完成评审 | 学生飞书收到 Bot 私聊：教师终评结果（通过/需修改 + 分数 + 评语） | ☐ |
| 5.3 | 教师发布新 Challenge | 班级群收到 Bot 公告（需配置 `FEISHU_CLASS_CHAT_ID`） | ☐ |
| 5.4 | 学生无 `feishu_open_id` | 通知跳过，飞书 AuditLogs 记录 `notify_failed` | ☐ |

### 第 6 步：权限验证（T9/T10）
| # | 操作 | 预期结果 | 通过？ |
|---|------|---------|--------|
| 6.1 | 学生 A 看 `/teacher` | 403 拒绝（非教师） | ☐ |
| 6.2 | 学生 A 看自己的提交详情 | 正常显示 | ☐ |
| 6.3 | 学生 A 看学生 B 的提交详情 | 403 "无权查看此提交" | ☐ |
| 6.4 | 清 Cookie 后直接访问 API | 401 未登录 | ☐ |
| 6.5 | 恶意修改 Cookie 中 role 字段 | HMAC 验签失败 → 401 | ☐ |

### 第 7 步：数据一致性
| # | 操作 | 预期结果 | 通过？ |
|---|------|---------|--------|
| 7.1 | Submissions 表 task_state 列存在 | T11 评审不降级（审计中无 `task_state_field_missing`） | ☐ |
| 7.2 | Submissions 表无脏记录 | `student_id` 不为空 | ☐ |

## 验收判定

- **通过**：全部 ☐ 打勾
- **有条件通过**：第 5 步中开放 ID 未填导致通知降级（属环境问题非代码问题）
- **不通过**：任一步骤出现 50x 错误、数据不一致、权限绕过
